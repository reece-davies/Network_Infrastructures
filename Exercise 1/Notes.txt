Dockerfile makes the image (page 3 figure 4) and docker compose takes that image and makes a new container.

Docker “build” creates a new image 	Docker “run” creates a new container (page 2)

//////////////////
/// Dockerfile ///
//////////////////
- Make new directory
- Go into the directory
- Make dockerfile (not sure why) (has FROM... & COPY...)
- call "$sudo docker build -t <image name> ."
- (OPTIONAL - making container) call "$sudo docker run -it -d --name <container name> -p 8080:80 <image name>"

^ Can all be done with a shell script
^ "$sudo docker image ls" can be used to view all created images

//////////////////////
/// Docker Compose ///
//////////////////////
- call "$nano docker-compose.yml" (creates docker compose file)
	- Paste docker compose content into this file
- Make directory "websites"
- Go into the directory
- Create index.html page ("Hello world from Docker Compose")
- Call "$sudo docker-compose up -d" (creates new docker container with the image specified in the docker-compose.yml file and looks at website html pages in websites folder)
	- A new container should have been created ("$sudo docker ps" or "$sudo docker ps -a")
